# VIBE TRAVEL 03 — Полный план разработки (MVP, визуальные улучшения, экспериментальные фичи)

## 1. Контекст и цели
- **Конкурс:** «Создай свой турсайт» (БГСХА, 2026).
- **Номинации:** Лучший дизайн-концепт и Лучшая подборка маршрутов (упор на дизайн).
- **Цель:** Создать статичный информационный сайт о молодёжном туризме в Бурятии.
- **Срок сдачи:** 30 апреля 2026.
- **Разработчик:** Студент, уверенно работает с кодом, ограничен по времени (~15 ч/нед).

## 2. Технологический стек (только разрешённое)
- HTML5 (семантика)
- CSS3 (Grid, Flexbox, CSS Variables, BEM)
- JavaScript (ES6+, Vanilla, без фреймворков)
- Библиотеки (через CDN): Yandex Maps API, jsPDF, html2canvas (опционально)
- Данные: JSON (`data/routes.json`, `data/events.json`)
- Шрифты: системные sans-serif (с возможностью замены на локальные .woff2)

## 4. Функциональные блоки (MVP)
| Страница | Обязательные элементы |
|----------|----------------------|
| **index.html** | Hero с лозунгом «Бурятия. Найди свой вайб», 3 блока-индикатора (заглушки), топ-5 маршрутов из JSON, превью карты (статичная картинка) |
| **routes.html** | Карточки маршрутов из JSON, фильтры по длительности, типу, сезону, бюджету (AND-логика), кнопка «В избранное» (localStorage) |
| **route-detail.html** | Галерея, описание, интерактивная карта с точками маршрута, практическая информация, пасхалка (blur-блок), кнопка скачивания PDF (текст) |
| **favorites.html** | Список избранных маршрутов с возможностью удаления |
| **map.html** | Яндекс.Карта с метками всех 10 маршрутов |
| **content.html** | Календарь событий (3 события из JSON) и 2 встроенных YouTube Shorts |
| **admin.html** | Скрытая страница с формой для генерации JSON-фрагмента маршрута |

## 5. Ключевые технические решения
- Фильтры и сортировка реализованы через JS (`Array.filter`, `Array.sort`).
- Кнопка «Скачать PDF» генерирует документ с названием, описанием и списком точек (без скриншота).
- Избранное хранится в `localStorage` (массив ID).
- Яндекс.Карта на детальной странице отображает полилинию маршрута и метки.
- Админ-панель: HTML-форма, вывод JSON в `<textarea>` для копирования. Защита: ссылка на `admin.html` не публикуется.

## 6. Дизайн-система (база)
- **Цвета:** Байкал (`#0077BE`), Кедр (`#2E8B57`), Золото (`#FFD700`), Фон (`#F5F0E6`).
- **Градиенты карточек:** Уникальные пары цветов из JSON.
- **Орнаменты:** SVG-паттерны в Hero и футере.
- **Анимации:** Только hover-эффекты и плавные переходы.

## 7. Этапы выполнения (для последовательной генерации кода)
1. **Этап 1:** `index.html`, базовые стили, `routes.json` с 10 маршрутами.
2. **Этап 2:** `routes.html` с фильтрами и карточками, `favorites.html`, логика избранного.
3. **Этап 3:** `route-detail.html`, Яндекс.Карта, PDF (текст).
4. **Этап 4:** `map.html`, `content.html`, `admin.html`.
5. **Этап 5:** Замена заглушек на фото, финальная оптимизация.

## 8. Визуальные улучшения (расширенный план)
Дополнительные визуальные фишки, которые можно внедрять по мере готовности основного функционала.

### 🎨 Анимации при скролле (Scroll Reveal)
- Используем `Intersection Observer` для добавления классов `.visible` элементам.
- Эффекты: плавное появление снизу/слева/справа, масштабирование.
- Реализуется в отдельном файле `src/js/animations.js`.

### 🎨 Микроанимации для вовлечения
- **Гастро-маршруты:** При наведении на карточку появляется лёгкая анимация «пара» (псевдоэлемент `::after` с blur).
- **Карточки маршрутов:** Эффект приподнимания и углубления тени при наведении.
- **Кнопки:** Плавное изменение фона и лёгкая пульсация (`box-shadow` с анимацией).

### 🎨 Улучшенные градиенты и цветовые акценты
- Динамическая смена фона Hero-секции в зависимости от времени суток (через CSS переменные, без JS).
- Использование конических градиентов для декоративных элементов.

### 🎨 Параллакс-эффект в Hero
- Фоновое изображение смещается медленнее при скролле (чистый CSS `background-attachment: fixed` с фолбэком).

### 🎨 Интерактивная карта-превью на главной
- Вместо статичной картинки — упрощённая Яндекс.Карта без контролов, только с метками (инициализируется лениво).

### 📅 Календарь событий с интерактивностью
- При клике на событие появляется модальное окно с подробным описанием.

### 📄 PDF со скриншотом карты (опционально)
- Добавляем библиотеку `html2canvas` (CDN).
- При генерации PDF делаем скриншот блока с картой и вставляем его в документ.
- **Важно:** Требует отладки из-за асинхронной загрузки карты. Рекомендуется сначала реализовать текстовый вариант, а эту функцию оставить на финал.

### 🛡️ Защита админ-панели
- Простой пароль через `prompt` при загрузке `admin.html`.
- Логин/пароль захардкожены в JS (для конкурса достаточно).

### 📝 Дополнительные разделы в README
- Инструкция по замене шрифтов на локальные `.woff2`.
- Рекомендации по сжатию изображений (TinyPNG) и улучшению Lighthouse.

## 9. Экспериментальные функции (продвинутый план)
Идеи, которые либо сложны в реализации, либо требуют больше времени, но могут сильно выделить проект.

### 🧪 1. Полноценная админ-панель с записью в файл (через сервер)
- **Проблема:** На чистом фронтенде нельзя записать JSON-файл на сервер.
- **Решение для конкурса:** Создаём простой Node.js сервер (Express) с одним эндпоинтом, который принимает POST-запрос с данными и перезаписывает `routes.json`. Админ-панель отправляет данные на этот сервер.
- **Требования:** Возможность запуска сервера локально или на хостинге (например, Glitch, Heroku бесплатный тир).
- **Преимущество:** Показывает жюри понимание fullstack-разработки.

### 🧪 2. Генерация PDF с векторной картой и таймлайном
- Вместо скриншота карты, используем библиотеку `leaflet` + `leaflet-image` для создания изображения карты программно.
- Или интеграция с Static Map API Яндекса (более надёжно). Формируем URL статической карты с маршрутом и вставляем в PDF.

### 🧪 3. Система «Вайб-фильтров» на основе машинного обучения (концепт)
- Добавить в пояснительную записку описание гипотетической рекомендательной системы, которая подбирает маршруты на основе настроения пользователя.
- Реализовывать не обязательно, но упомянуть как видение проекта.

### 🧪 4. PWA и офлайн-доступ
- Добавить Service Worker для кэширования страниц и JSON.
- Сайт можно будет открывать без интернета (после первого посещения).
- Увеличивает баллы по критерию «инновационность».

### 🧪 5. Кастомный плеер для видео с эффектами
- В разделе «Контент» вместо обычного iframe — стилизованный видеоплеер с анимацией бурятских узоров по краям.

### 🧪 6. Геймификация
- Система «Достижений»: пользователь получает бейджи за просмотр маршрутов или добавление в избранное (хранятся в `localStorage`).

### ⚠️ Важно
Эти фичи не должны ставить под угрозу сдачу базового функционала. Их стоит рассматривать как бонусные, если останется время и желание поэкспериментировать.
